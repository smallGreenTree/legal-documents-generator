## 4. Core Components – Detailed Specification

### 4.1 Tracer (`tasks/tracer.py`) – “Tracy for LLMs” (MUST be implemented first)

**Requirements:**
- Every single LLM call **must** go through the tracer
- Store full prompt + response + metadata as JSON
- Hierarchical tracing (parent_task_id)
- Easy to search/replay later

**Trace JSON schema (example):**
```json
{
  "trace_id": "uuid",
  "timestamp": "2026-04-07T11:52:30.123456",
  "doc_id": "en_indictment_financial_fraud_002",
  "task_id": "writer_facts_chunk_03",
  "parent_task_id": "planner_facts",
  "stage": "writer",
  "model": "qwen3:8b",
  "prompt": "...full prompt including memory...",
  "response": "...full LLM output...",
  "metadata": {
    "chunk_size": 650,
    "temperature": 0.6,
    "tokens_prompt": 12450,
    "tokens_response": 820,
    "latency_ms": 12400
  }
}
```

Implement functions:
- `start_trace(doc_id, task_id, parent_task_id=None)`
- `record_llm_call(trace_id, prompt, response, metadata)`
- `get_traces_for_doc(doc_id)` + simple viewer script

### 4.2 Memory Manager (`tasks/memory_manager.py`)

- Creates/updates `CASE_MEMORY.md` for each document
- Contains: schema summary, strict rules, style guide, running facts, consistency notes
- Injected at the **beginning** of every prompt

### 4.3 Planner (`tasks/planner.py`)

- Stage 1: Document-level planner (high-level outline + chronology)
- Stage 2: Per-section planner (bullet-point facts + entities to mention)

### 4.4 Writer (`tasks/writer.py`)

- Generates in small chunks (500–800 words)
- Always receives: full CASE_MEMORY + planner output + tail of previous chunk
- Strict instruction: “Use ONLY entities and facts from CASE_MEMORY.md”

### 4.5 Critic (`tasks/critic.py`)

- Reviews generated section against CASE_MEMORY
- Returns: list of issues + suggested fixes + revised text (if needed)

### 4.6 Orchestrator (`tasks/orchestrator.py`)

Main loop:
```python
1. Load/create CASE_MEMORY.md
2. Document planner
3. For each section:
   a. Section planner
   b. Writer (multiple chunks)
   c. Critic + revision loop (max 2 revisions)
   d. Validator
4. Final render with Jinja2
5. Write generation_report.md with all trace links
```
